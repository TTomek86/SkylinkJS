Before using Skylink

Please invoke Skylink/init:method method first to initialise the Application Key before using any functionalities in Skylink.

If you do not have an Application Key, you may register for a Skylink platform developer account to create one.

To get started you may visit the getting started page, or alternatively fork a ready made demo application that uses Skylink Web SDK at getaroom.io.

For tips on using skylink and troubleshooting you can visit our support portal.

Example:

        
  // Here's a simple example on how you can start using Skylink
  var SkylinkDemo = new Skylink();


  // Subscribe all events first before init()
  SkylinkDemo.on("incomingStream", function (peerId, stream, peerInfo, isSelf) {
    if (isSelf) {
      attachMediaStream(document.getElementById("selfVideo"), stream);
    } else {
      var peerVideo = document.createElement("video");
      peerVideo.id = peerId;
      peerVideo.autoplay = "autoplay";
      document.getElementById("peersVideo").appendChild(peerVideo);
      attachMediaStream(peerVideo, stream);
    }
  });


  SkylinkDemo.on("peerLeft", function (peerId, peerInfo, isSelf) {
    if (!isSelf) {
      var peerVideo = document.getElementById(peerId);
      // do a check if peerVideo exists first
      if (peerVideo) {
        document.getElementById("peersVideo").removeChild(peerVideo);
      } else {
        console.error("Peer video for " + peerId + " is not found.");
      }
    }
  });


 // never call joinRoom in readyStateChange event subscription.
 // call joinRoom after init() callback if you want to joinRoom instantly.
 SkylinkDemo.on("readyStateChange", function (state, room) {
   console.log("Room (" + room + ") state: ", room);
 })


 // always remember to call init()
 SkylinkDemo.init("YOUR_APP_KEY_HERE", function (error, success) {
   // do a check for error or success
   if (error) {
     console.error("Init failed: ", error);
   } else {
     SkylinkDemo.joinRoom("my_room", {
       userData: "My Username",
       audio: true,
       video: true
     });
   }
 });

ICE_CONNECTION_STATE

Attribute Type: {JSON}
readonly
  Defined in: source/constants.js:287      Available since 0.7.0

Contains the list of ICE connection states of a Peer connection.

Keys:

  • STARTING {String}

    Value "starting" The state when ICE agent is waiting for remote candidates supplied from ICE gathering from Peer.

  • CHECKING {String}

    Value "checking" The state when ICE agent has received the remote candidates from Peer and is checking for a usable candidate pair to start ICE connection.

  • CONNECTED {String}

    Value "connected" The state when ICE agent has found a usable connection but still checking if there is a better candidate pair for better ICE connection.
    At this stage, ICE connection is already established and may not necessarily go to COMPLETED.

  • COMPLETED {String}

    Value "completed" The state when ICE agent has finished checking for the best candidate pairs for the best ICE connection.
    At this stage, ICE connection is has been already established.

  • FAILED {String}

    Value "failed" The state when ICE agent had failed to find a ICE connection from all candidate pairs.

  • DISCONNECTED {String}

    Value "disconnected" The state when ICE agent connection is disconnected abruptly and may happen on a flaky network.

  • CLOSED {String}

    Value "closed" The state when ICE agent connection is closed and is no longer responding to any STUN requests.

  • TRICKLE_FAILED {String}

    Value "trickeFailed" The state when after ICE agent connection state had gone to FAILED and init() method enableIceTrickle option is enabled.

READY_STATE_CHANGE_ERROR

Attribute Type: {JSON}
readonly
  Defined in: source/constants.js:449      Available since 0.7.0

Note that for the next releases, the Keys and Values will change for a more descriptive term.
Contains the list of init() method SDK initialization errors.

Keys:

  • API_INVALID {Number}

    Value 4001 The error when provided Application Key is does not exists. Resolve this by finding if Application Key exists in your Skylink Developer Account.

  • API_DOMAIN_NOT_MATCH {Number}

    Value 4002 The error when accessing backend IP address does not match Application Key configured domain. Resolve this by seeking support in Temasys Support Portal.

  • API_CORS_DOMAIN_NOT_MATCH {Number}

    Value 4003 The error when accessing CORS domain does not match Application Key configured corsurl. Resolve this by configuring your Application Key CORS Url correctly in your Skylink Developer Account.

  • API_CREDENTIALS_INVALID {Number}

    Value 4004 The error when credentials provided for Application Key is invalid. Resolve this by checking if credentials are generated correctly, or Secret used for generating credentials is correct. See init() method on how to generate the credentials.

  • API_CREDENTIALS_NOT_MATCH {Number}

    Value 4005 The error when credentials provided for Application Key does not match provided duration and startDate in generated credentials. Resolve this by checking if credentials are generated correctly. See init() method on how to generate the credentials.

  • API_INVALID_PARENT_KEY {Number}

    Value 4006 The error when provided Application Key has an invalid parent field value. Resolve this by providing another Application Key. You may check if it's valid by accessing your Skylink Developer Account and use any Application Keys from the displayed list of Application Keys.

  • API_NO_MEETING_RECORD_FOUND {Number}

    Value 4010 The error when provided Persistent Room enabled key does not have any meeting records that matches the credentials provided in init() method. See Persistent Rooms and how it works.

  • NO_SOCKET_IO {Number}

    Value 1 The error when there is no socket.io-client dependency loaded before the SDK. Resolve this by loading the dependency based on the correct versions following SkylinkJS release versions.

  • NO_XMLHTTPREQUEST_SUPPORT {Number}

    Value 2 The error when there is no XMLHttpRequest API interface supported in the browser. Resolve this by switching to one of our supported browsers list.

  • NO_WEBRTC_SUPPORT {Number}

    Value 3 The error when there is no RTCPeerConnection API interface supported in the browser, which the Temasys Plugin does not support and is not installed in the browser. Resolve this by switching to one of our supported browsers list, or if browser is Safari or IE in our supported browsers list, prompt users to install the Temasys Plugin.

  • NO_PATH {Number}

    Value 4 The error when invalid init() method options are provided. Resolve this by ensuring correct init() method options are provided.

  • ADAPTER_NO_LOADED {Number}

    Value 7 The error when there is no adapterjs dependency loaded before the SDK. Resolve this by loading the dependency based on the correct versions following SkylinkJS release versions.

  • XML_HTTP_REQUEST_ERROR {Number}

    Value -1 The error when there is no response from server when requesting to start connection session. Resolve this by diagnosing network connections or refreshing the page. You may report this to Temasys Support Portal if both solutions does not work.
