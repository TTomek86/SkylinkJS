Before using Skylink

Please invoke init() method first to initialise the Application Key before using any functionalities in Skylink.

If you do not have an Application Key, you may register for a Skylink platform developer account to create one.

To get started you may visit the getting started page, or alternatively fork a ready made demo application that uses Skylink Web SDK at getaroom.io.

For tips on using skylink and troubleshooting you can visit our support portal.

Example:

        
  // Here's a simple example on how you can start using Skylink
  var SkylinkDemo = new Skylink();


  // Subscribe all events first before init()
  SkylinkDemo.on("incomingStream", function (peerId, stream, peerInfo, isSelf) {
    if (isSelf) {
      attachMediaStream(document.getElementById("selfVideo"), stream);
    } else {
      var peerVideo = document.createElement("video");
      peerVideo.id = peerId;
      peerVideo.autoplay = "autoplay";
      document.getElementById("peersVideo").appendChild(peerVideo);
      attachMediaStream(peerVideo, stream);
    }
  });


  SkylinkDemo.on("peerLeft", function (peerId, peerInfo, isSelf) {
    if (!isSelf) {
      var peerVideo = document.getElementById(peerId);
      // do a check if peerVideo exists first
      if (peerVideo) {
        document.getElementById("peersVideo").removeChild(peerVideo);
      } else {
        console.error("Peer video for " + peerId + " is not found.");
      }
    }
  });


 // never call joinRoom in readyStateChange event subscription.
 // call joinRoom after init() callback if you want to joinRoom instantly.
 SkylinkDemo.on("readyStateChange", function (state, room) {
   console.log("Room (" + room + ") state: ", room);
 })


 // always remember to call init()
 SkylinkDemo.init("YOUR_APP_KEY_HERE", function (error, success) {
   // do a check for error or success
   if (error) {
     console.error("Init failed: ", error);
   } else {
     SkylinkDemo.joinRoom("my_room", {
       userData: "My Username",
       audio: true,
       video: true
     });
   }
 });

init

( options ,   [callbackFn] ,   )
required
  Available since 0.7.0

Note that this method is required to call before using any other functionalities except Event methods like on() method, off() method or once() method. By default, the SDK uses CORS authentication. To use credentials based authentication, configure the credentials parameter.
Method that initializes the Skylink object to start connection session.

Parameters:

  • options {String | JSON}

    The parameter for configuration settings.
    Providing this value as a "String" reads as appKey.

    • appKey {String}

      The parameter to configure the Application Key.

    • defaultRoom {String} Optional
      DEFAULT: &options.appKey

      This fallbacks to the value of Application Key configured if not provided.
      The parameter to configure the default Room name that the SDK uses when no room parameter is provided in joinRoom() method.

    • roomServer {String} Optional

      This is for SDK debugging purposes and it is NOT RECOMMENDED to modify the value. To use this, have to provide the Protocol Relative URL value ("//") before the domain name. Ensure that no ("/") value is provided at the end of the value.
      The parameter to configure the API server to start connection session.

    • region {String} Optional

      This has been deprecated and it is not recommended to configure this value anymore.
      The parameter to configure the nearest regional server for the SDK to connect to. [Rel: Skylink.attr.REGIONAL_SERVER]

    • enableIceTrickle {Boolean} Optional
      DEFAULT: true

      Note that setting this to false may result in a slower connection.
      The parameter to enable trickle ICE connections for faster connectivity. See: What is ICE and how it works

    • enableDataChannel {Boolean} Optional
      DEFAULT: true

      Note that setting this to false will disable sendBlobData() method, sendURLData() method and sendP2PMessage() method functionalities. You will not be able to receive any P2P messages as well. Additionally, even with this enabled, if there is a connecting Peer that does not have enableDataChannel enabled, the following above functionalities will not work with that particular Peer connection.
      The parameter to enable Datachannel connections.

    • enableTURNServer {Boolean} Optional
      DEFAULT: true

      Note that this only filters the ICE servers when starting Peer connection. To enforce TURN connections, configure forceTURN parameter instead.
      The parameter to enable using TURN ICE servers when starting a Peer connection.
      Disabling this will filter out all TURN ICE servers passed in Peer connections.

    • enableSTUNServer {Boolean} Optional
      DEFAULT: true

      The parameter to enable using STUN ICE servers when starting a Peer connection.
      Disabling this will filter out all STUN ICE servers passed in Peer connections.

    • forceTURN {Boolean} Optional
      DEFAULT: false

      Note that if TURN is not enabled for the Application Key configured, Peer connections will not be able to connect.
      The parameter to enforce TURN ICE connections.
      Enabling this will filter out all STUN ICE servers as well.

    • usePublicSTUN {Boolean} Optional
      DEFAULT: true

      The parameter to enable using publicly available STUN ICE servers when starting a Peer connection.
      Disabling this will filter out all publicly available STUN ICE servers.

    • TURNServerTransport {Boolean} Optional
      DEFAULT: #<Skylink.TURN_TRANSPORT.ANY>

      The parameter to configure the TURN ICE servers transports. [Rel: Skylink.attr.TURN_TRANSPORT]

    • credentials {JSON} Optional

      Note that you require the configure the correct startDateTime and duration used when generating the credentials to configure the SDK to use credentials based authentication, By default, when credentials is not provided, the SDK will use CORS based authentication. Additionally, take note that using credentials based authentication does not allow you to join the Room based on the room paramter provided in joinRoom() method.
      The parameter to use credentials based authentication to start connection session. See: Authenticating SDK connection sessions.

      • startDateTime {String}
        The parameter to configure the credentials starting DateTime. This value has to be set in (ISO 8601 format), which can be retrieved from the Date object Date.getISOString() method.
      • duration {Number}
        The parameter to configure the credentials connection session duration in Hours. EXAMPLES:
        30 mins: 0.5
        1 hour, 24mins: 1.4
        2 hours, 30mins: 2.5
      • credentials {String}
        The parameter to configure the credentials connection session authentication string.
        This requires you to generate with your configured Application Key Secret, provided credentials - startDateTime, duration and defaultRoom values. INSTRUCTIONS ON GENERATING CREDENTIALS STRING:
        1. Concatenate the defaultRoom, credentials - duration and startDateTime. var concatStr = defaultRoom + "_" + duration + "_" + startDateTimeStamp;
        2. Hash the concatStr value with the Application Key Secret using SHA-1 In this example, we use the CryptoJS.HmacSHA1 function . var hash = CryptoJS.HmacSHA1(concatStr, secret);
        3. Convert the hash to a Base64 encoded string. In this example, we use the CryptoJS.enc.Base64 function. var base64String = hash.toString(CryptoJS.enc.Base64);
        4. Encode the Base64 encoded string to a URI component using UTF-8 encoding with encodeURIComponent() function. var credentialsStr = encodeURIComponent(base64String);
    • audioFallback {Boolean} Optional
      DEFAULT: false

      The parameter to configure to retrieve audio only as a fallback when retrieving audio and video fails in getUserMedia() method.

    • forceSSL {Boolean} Optional
      DEFAULT: false

      By default, if user is accessing the SDK from https: protocol, SSL connections are already enabled.
      The parameter to enforce SSL connections when starting connection session and Room socket connection.

    • audioCodec {String} Optional
      DEFAULT: #<Skylink.AUDIO_CODEC.AUTO>

      The parameter to configure the audio codec for audio streams. [Rel: Skylink.attr.AUDIO_CODEC]

    • videoCodec {String} Optional
      DEFAULT: #<Skylink.VIDEO_CODEC.AUTO>

      The parameter to configure the audio codec for video streams. [Rel: Skylink.attr.VIDEO_CODEC]

    • socketTimeout {Number} Optional
      DEFAULT: 20000

      The parameter to configure the timeout to wait when there is no connection response before triggering as CONNECTION_FAILED or RECONNECTION_FAILED. [Rel: Skylink.attr.SOCKET_ERROR]

    • forceTURNSSL {Boolean} Optional
      DEFAULT: false

      By default, if user is accessing the SDK from https: protocol, TURN SSL connections are already enabled. Note that if the browser does not support the turns: protocol, SSL ports will be used instead.
      The parameter to enforce SSL connections for TURN ICE server connections.

  • callbackFn {Function} Optional

    The parameter callback function that would be triggered when after init() method process has completed.

    Method parameter payload signature: function(error, success)

    • error {JSON}

      The callback function parameter payload that is defined when init() method process encountered an error.

      • errorCode {Number}
        The error code. [Rel: Skylink.attr.READY_STATE_CHANGE_ERROR]
      • error {Error}
        The error object.
      • status {Number}
        The HTTP status received when encountering the error.
        [Rel: Skylink.attr.READY_STATE_CHANGE_ERROR_STATUS]
    • success {JSON}

      The callback function parameter payload that is defined when init() method process has completed without errors.

      • serverUrl {String}
        This has been deprecated and will be removed in the next releases.
        The API HTTP request URL.
      • readyState {String}
        This has been deprecated and will be removed in the next releases.
        The current connection session ready state.
      • appKey {String}
        The configured parameter appKey value.
      • defaultRoom {String}
        The configured parameter defaultRoom value.
      • selectedRoom {String}
        This has been deprecated in the callback and will be removed in the next releases.
        The current selected Room name.
      • roomServer {String}
        The configured parameter roomServer value.
      • enableIceTrickle {Boolean}
        The configured parameter enableIceTrickle value.
      • enableDataChannel {Boolean}
        The configured parameter enableDataChannel value.
      • enableTURNServer {Boolean}
        The configured parameter enableTURNServer value.
      • enableSTUNServer {Boolean}
        The configured parameter enableSTUNServer value.
      • TURNTransport {Boolean}
        This will be renamed as TURNServerTransport in the next releases.
        The configured parameter TURNServerTransport value.
      • serverRegion {String} Optional
        This will be renamed as region in the next releases before removal of region parameter.
        The configured parameter region value.
      • audioFallback {Boolean}
        The configured parameter audioFallback value.
      • forceSSL {Boolean}
        The configured parameter forceSSL value.
      • audioCodec {String}
        The configured parameter audioCodec value.
      • videoCodec {String}
        The configured parameter videoCodec value.
      • socketTimeout {Number}
        The configured parameter socketTimeout value.
      • forceTURNSSL {Boolean}
        The configured parameter forceTURNSSL value.
      • forceTURN {Boolean}
        The configured parameter forceTURN value.
      • usePublicSTUN {Boolean}
        The configured parameter usePublicTURN value.

off

( [eventName] ,   [listenerFn] ,   )
  Available since 0.7.0

Method that unsubscribes listener functions from Events.

Parameters:

  • eventName {String} Optional

    Not providing this parameter value removes all listener functions from all Events.
    The parameter name of the Event to unsubscribe listener functions.

  • listenerFn {Function} Optional

    Not providing this parameter value removes all listener functions from the provided eventName parameter Event.
    The parameter of a particular listener function to unsubscribe from the provided eventName parameter Event.

Example:

        
  // Parameters: off(eventName, listenerFn)
  //! Unsubscribes this specific listener function from "iceConnectionState" Event
  var listenerFn = function (state, peerId) {
    console.log(peerId + " connection state: " + state);
  };


  SkylinkDemo.off("iceConnectionState", listenerFn);


  // Parameters: off(eventName)
  //! Unsubscribes all listener functions from "peerJoined" Event
  SkylinkDemo.off("peerJoined");


  // Parameters: off()
  //! Unsubscribes all listener functions from all Events


  SkylinkDemo.off();

once

( eventName ,   listenerFn ,   [conditionalFn] ,   )
  Available since 0.7.0

Method that subscribes a listener function to an Event which would be triggered only once when the Event is dispatched.

Parameters:

  • eventName {String}

    The parameter name of the Event to subscribe to.

  • listenerFn {Function}

    The parameter listener function to bind to the subscription.

  • conditionalFn {Function} Optional

    This function will trigger every time the Event is dispatched. Return true in this function to complete the conditional function.
    The parameter conditional function that halts the triggering of the listener function until the conditional function returns true.

Example:

        
  // Parameters: once(eventName, listenerFn)
  SkylinkDemo.on("mediaAccessSuccess", function (stream) {
    console.log("Received local stream", stream);
    ///-[Handle UI view when user has given access to local Stream]
  });


  // Parameters: once(eventName, listenerFn, conditionalFn)
  SkylinkDemo.on("dataChannelState", function (state, peerId) {
    console.info("P2P messaging are now available with " + peerId);
    ///-[Handle UI view when P2P messaging is available]
  }, function (state, peerId, error, channelName, channelType) {
    //! Condition to ensure that Datachannel is opened and is for messaiging Datachannel
    return state === SkylinkDemo.DATA_CHANNEL_STATE.OPEN &&
      channelType === SkylinkDemo.DATA_CHANNEL_TYPE.MESSAGING;
  });

ICE_CONNECTION_STATE

Attribute type is JSON
readonly
  Available since 0.7.0

Contains the list of ICE connection states of a Peer connection.

Keys:

  • STARTING {String}

    Value "starting" The state when ICE agent is waiting for remote candidates supplied from ICE gathering from Peer.

  • CHECKING {String}

    Value "checking" The state when ICE agent has received the remote candidates from Peer and is checking for a usable candidate pair to start ICE connection.

  • CONNECTED {String}

    Value "connected" The state when ICE agent has found a usable connection but still checking if there is a better candidate pair for better ICE connection.
    At this stage, ICE connection is already established and may not necessarily go to COMPLETED.

  • COMPLETED {String}

    Value "completed" The state when ICE agent has finished checking for the best candidate pairs for the best ICE connection.
    At this stage, ICE connection is has been already established.

  • FAILED {String}

    Value "failed" The state when ICE agent had failed to find a ICE connection from all candidate pairs.

  • DISCONNECTED {String}

    Value "disconnected" The state when ICE agent connection is disconnected abruptly and may happen on a flaky network.

  • CLOSED {String}

    Value "closed" The state when ICE agent connection is closed and is no longer responding to any STUN requests.

  • TRICKLE_FAILED {String}

    Value "trickeFailed" The state when after ICE agent connection state had gone to FAILED and init() method enableIceTrickle option is enabled.

  Defined in: source/events/core.js:2      Available since 0.4.0

Event triggered when room connection information is being retrieved from platform server.

  • This is also triggered when init() is invoked, but the socket connection events like channelOpen does not get triggered but stops at readyStateChange event.

Event Payload:

  • readyState {String}

    The current ready state of the retrieval when the event is triggered. [Rel: Skylink.READY_STATE_CHANGE]

  • error {JSON} Optional

    The error object thrown when there is a failure in retrieval. If received as null, it means that there is no errors.

    • status {Number}

      Http status when retrieving information. May be empty for other errors.

    • errorCode {Number}

      The READY_STATE_CHANGE_ERROR if there is an readyStateChange event error that caused the failure for initialising Skylink. [Rel: Skylink.READY_STATE_CHANGE_ERROR]

    • content {Object}

      The exception thrown that caused the failure for initialising Skylink.

    • status {Number}

      The XMLHttpRequest status code received when exception is thrown that caused the failure for initialising Skylink.

  • room {String}

    The selected room connection information that Skylink is attempting to retrieve the information for to start connection to.
